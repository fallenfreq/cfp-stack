Block Builder

A rich content editor for Vue 3, built on Tiptap and ProseMirror. Every block is draggable, every node is configurable, and layouts compose without writing CSS. This page is built entirely with Block Builder — explore it to see what's possible.

Quick Reference

  • Type / to open the command palette

  • Drag the ⠿ handle to reorder blocks

  • Select text or click a block to open the floating toolbar

  • Click the node path breadcrumb to set drag depth and toolbar target

  • Click < > at the bottom right to toggle HTML source view

Layout Components

Layout components are block containers inserted via the / command. They control arrangement without touching CSS, and all support the global attribute editor for custom style, class, and id overrides.

LayoutSection

A flex-column container with configurable gap, padding, and align props. Use it to group blocks with consistent spacing. The three paragraphs below are inside a LayoutSection with gap="sm" and align="start".

LayoutSplit

Creates a two-column asymmetric grid. The split prop sets the ratio: 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, or 3/4. The two cards below are the two direct children of this LayoutSplit — each becomes one grid cell. The sf-collapse-sm class sets the container-query breakpoint at which it stacks to a single column.

Left column

Any block content can go inside — paragraphs, lists, nested layout components, images, code blocks. This is a split="1/2" (equal width) layout.

Right column

The sf-gap-* class sets the gutter. Try the attribute editor on the LayoutSplit wrapper to change the split ratio or gap live.

LayoutColumns

An N-column equal grid — 2, 3, or 4 columns. Each direct child becomes one cell. The three cards below are in a columns="3" grid.

Column 1

2, 3, or 4 equal columns. Each direct child is one cell.

Column 2

The gap prop controls the gutter between columns.

Column 3

The sf-collapse-sm class sets the breakpoint at which it stacks to one column.

LayoutCard

A styled container with five variants: elevated (box shadow, the default), outlined (border only), filled (secondary background), feature (primary-tinted), and plain (no decoration). Padding and radius are set via sf-padding-* and sf-radius-* classes.

LayoutCenter

Constrains children to a max-width and centres them horizontally. Options: xs (20rem) through 3xl (48rem) or full. This page uses it as the top-level reading column — full-bleed blocks like the hero image and the code sample below sit outside a LayoutCenter at the document root.

Text & Block Formatting

Select any text to reveal the floating toolbar. Inline formatting works on any text node: bold, italic, strikethrough, inline code, and links. Standard keyboard shortcuts apply — Ctrl+B, Ctrl+I — as do markdown input rules like **bold** and `code`. Click the link above or select text and use the link toolbar icon to set a URL.

Blockquotes are available from the slash command, the Turn Into picker, or Wrap In to wrap any existing block. Start a line with > as a markdown shortcut.

Block Types

Every block type is available from the / command palette. Headings, lists, and blockquotes also have markdown shortcuts that fire as you type. Horizontal rules are available via /hr or by typing --- on a blank line.

Code Blocks

Syntax highlighting for JavaScript, TypeScript, Python, JSON, CSS, and HTML. Select auto for automatic detection. When a code block is focused the toolbar shows a auto_fix_high Format button — for JavaScript and HTML it runs a full code formatter in place. The block below is a sibling of this LayoutCenter at the document root, so it stretches edge-to-edge. 

import { useEditor } from '@tiptap/vue-3'
import { blockBuilderExtensions } from '@/editor/extensions'

const editor = useEditor({
  extensions: blockBuilderExtensions({
    dragHandle: true,
    multiSelect: true,
    floatingToolbar: true,
  }),
  content: initialContent,
})

The Floating Toolbar

The toolbar appears above any selected node or text range and adapts its controls to what makes sense in context. Hovering over a button shows a tooltip. Every icon below is the actual button you will see in the toolbar.

Block Conversion

Available on most block nodes:

  • change_circle Turn Into — convert to heading, paragraph, blockquote, list, or code block

  • frame_source Wrap In — wrap the node inside any layout container (block content containers only)

  • move_up Unwrap — remove the wrapper and promote its children

  • delete Delete Node — remove the selected block entirely

Inline Formatting

Shown when cursor is in a text block:

  • format_bold Bold — Ctrl+B

  • format_italic Italic — Ctrl+I

  • format_strikethrough Strikethrough

  • code Inline code

  • link Link — reveals a URL input; click an existing link to edit or remove it

Attribute Editor

Available on every node at depth > 0. tune opens a panel where you can set:

  • style — inline CSS with a dedicated style editor

  • class — CSS class list

  • id — element ID

  • Component props — gap, padding, variant, split ratio, etc., shown as dropdowns for enum values

Table Controls

Appear only when the cursor is inside a table cell:

  • + Row ↑ / + Row ↓ — add a row above or below

  • Del Row — delete the current row

  • + Col ← / + Col → — add a column left or right

  • Del Col — delete the current column

List Controls

Appear when the active node is a list wrapper:

  • format_list_bulleted Toggle bullet list

  • format_list_numbered Toggle ordered list

  • checklist Toggle task list

  • Switching between types rewrites item node types and preserves content

Media & Code Controls

  • edit Image URL — shown when an image is selected; reveals a URL input to update the src

  • edit YouTube URL — shown when a video is selected; update the embed URL

  • auto_fix_high Format code — shown for code blocks; runs the formatter for JS and HTML

Code View

The < > button at the bottom-right of the editor toggles between the visual editor and raw HTML source. Edits made in code view are parsed back into the visual editor on toggle. Useful for pasting markup directly or inspecting the generated HTML. Most toolbar actions are hidden in code view since they operate on the visual document model.

Node Path & Depth Control

The Node Path Breadcrumb

At the top of the editor a breadcrumb shows the full ancestor path of the cursor — for example: doc › LayoutCard › p. Clicking any segment sets the active depth to that level, which controls two things simultaneously:

  • Toolbar target — the toolbar targets the shallowest ancestor at or below the active depth. At depth 1 it targets the top-level block; at depth 2 it targets the nested block the cursor is inside.

  • Drag handles — every block at depth ≤ active depth shows a handle. The number below each handle shows its depth level.

How the Depth Ceiling Works

Active depth is a ceiling, not an exact level. At active depth 2, blocks at depth 1 and depth 2 both get handles.

  • Depth 1 (default) — only top-level blocks are targetable and draggable

  • Depth 2 — top-level blocks keep their handles; their direct block children gain handles too. A depth-1 block that contains depth-2 children shows handles at both levels — the outer handle is accessible from the margin area around the block.

  • A depth-1 block with no depth-2 children appears with a single handle and is straightforward to drag at any active depth.

Drag Handle & Multi-select

Drag Handle

A ⠿ handle appears to the left of any draggable block when you hover near it. Drag it to reorder. The handle fades after 2 seconds of inactivity — move the cursor back to the left edge to show it again.

Which blocks show handles is determined by the active depth. At depth 1 only the outermost blocks are draggable; clicking deeper breadcrumb segments makes nested blocks draggable too.

Multi-select (Virtual Selection)

When a selection is active the toolbar switches to virtual-selection mode:

  • check_box_outline_blank / check_box Toggle — add or remove the current node

  • keyboard_arrow_up keyboard_arrow_down Add prev / next sibling

  • select_all Add all siblings · unfold_more Add all children

  • vertical_align_top vertical_align_bottom Move before / after target

  • wrap_text Wrap selection · find_replace Replace target

  • delete Delete selected · close Clear

Dragging a node in a multi-select moves the entire group.
