Welcome to the Interactive Editor!

This is a demo showcasing the flexibility of the editor. You can use registered Vue components, apply custom styles and classes, embed YouTube videos, display images, add tables, and even work with syntax-highlighted code blocks.

The editor allows you to switch between a WYSIWYG view and an editable code view. This means you can directly modify the content or structure as code and see the changes live. Note that the purpose here is to show the code that generates the final HTML output rather than the output itself.

Vuestic (component library) Buttons

The following example shows two buttons rendered using the <VaButton> component from the Vuestic UI library. You can apply custom classes and styles to modify their appearance. In this case, we're using inline styles and classes to demonstrate how you can customise the look of these buttons.

A limitation of the editor is that an element can only be set to contain either block or inline elements, and a single element cannot contain a mix. The button below, which is a Vue component, needs to contain inline elements but the first button contains a block element. The block element is automatically removed, so we cannot see the pink.

Button contains block el

The component contains an inline element which is rendered as intended. If the component's content was supposed to be block, the inline element would be wrapped in a <p> tag instead of being removed like with the block example.

JavaScript Code with Syntax Highlighting

The editor also supports syntax highlighting for code and automatically highlights the syntax based on the language selected — or detected if auto is selected. JavaScript and HTML also support formatting by pressing the `Format` button while the cursor is in the code block. 

// Simple JavaScript FizzBuzz example
for (var i = 1; i <= 20; i++) {
  if (i % 15 === 0) {
    console.log("FizzBuzz");
  } else if (i % 3 === 0) {
    console.log("Fizz");
  } else if (i % 5 === 0) {
    console.log("Buzz");
  } else {
    console.log(i);
  }
}

Markdown Shortcuts

Markdown shortcuts make it easy to format the text while typing.

To test that, start a new line and type # followed by a space to get a heading. Try #, ##, ###, ####, #####, ###### for different levels.

These conventions are called input rules in Tiptap. Some of them are enabled by default. Try > for blockquotes, *, - or + for bullet lists, or `foobar` to highlight code, and ~~tildes~~ to strike text.
