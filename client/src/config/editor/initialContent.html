Block Builder

A rich content editor for Vue 3, built on Tiptap and ProseMirror. Every block is draggable, every node is configurable, and layouts compose without writing CSS. This page is built entirely with Block Builder — explore it to see what's possible.

Quick Reference

  • Type / to open the command palette

  • Drag the ⠿ handle to reorder blocks

  • Select text to open the floating toolbar

  • Click the node path breadcrumb to control drag depth

Layout Components

Layout components are block containers inserted via the / command. They control arrangement without touching CSS, and all support the global attribute editor for custom style, class, and id overrides.

LayoutSection

A flex-column container with configurable gap, padding, and align props. Use it to group blocks with consistent spacing. The three paragraphs below are inside a LayoutSection with gap="sm" and align="start".

LayoutSplit

Creates a two-column asymmetric grid. The split prop sets the ratio: 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, or 3/4. The two cards below are the two direct children of this LayoutSplit — each becomes one grid cell. The collapse prop controls the container-query breakpoint at which it stacks to a single column.

Left column

Any block content can go inside — paragraphs, lists, nested layout components, images, code blocks. This is a split="1/2" (equal width) layout.

Right column

The gap prop sets the gutter. Try the attribute editor on the LayoutSplit wrapper to change the split ratio or gap live.

LayoutColumns

An N-column equal grid — 2, 3, or 4 columns. Each direct child becomes one cell. The three cards below are in a columns="3" grid.

Column 1

2, 3, or 4 equal columns. Each direct child is one cell.

Column 2

The gap prop controls the gutter between columns.

Column 3

The collapse prop sets the breakpoint at which it stacks to one column.

LayoutCard

A styled container with four variants: elevated (box shadow, the default), outlined (border), filled (secondary background), and plain (no decoration). Also accepts padding and radius props.

LayoutCenter

Constrains children to a max-width and centres them horizontally. Options: xs (20rem) through 2xl (42rem) or full. Used below for the code example.

Text Formatting

Select any text to reveal the floating toolbar. Inline formatting is available on any text node: bold, italic, strikethrough, and inline code. Standard keyboard shortcuts work too — Ctrl+B, Ctrl+I — as do markdown input rules like **bold** and `code`.

Blockquotes are available from the slash command or by converting a paragraph via the toolbar's Turn Into picker. Start a line with > as a markdown shortcut.

Block Types

Every block type is available from the / command palette. Headings, lists, and blockquotes also have markdown shortcuts that fire as you type.

Lists

Three list types — bullet, ordered, and task. All can be converted between each other from the toolbar. Markdown shortcuts: - or * for bullet, 1. for ordered.

Code Blocks

Syntax highlighting for JavaScript, TypeScript, Python, JSON, CSS, and HTML. Select auto for automatic detection. When a code block is focused the toolbar shows a Format button — for JavaScript and HTML it runs a full code formatter in place. 

import { useEditor } from '@tiptap/vue-3'
import { blockBuilderExtensions } from '@/editor/extensions'

const editor = useEditor({
  extensions: blockBuilderExtensions({
    dragHandle: true,
    multiSelect: true,
    floatingToolbar: true,
  }),
  content: initialContent,
})

The Floating Toolbar

The toolbar appears above any selected node or text range and adapts its items to what makes sense in context. Hovering over a button shows a tooltip describing what it does.

Block Conversion

  • Turn Into — convert the block to a different type: heading levels, paragraph, blockquote, bullet list, ordered list, code block

  • Wrap In — wrap the selection inside a container: any layout component or other block wrapper

  • Unwrap — remove the wrapper node and promote its children to the parent level

  • Delete Node — remove the selected block entirely

Inline Formatting

Shown when any text is selected:

  • Bold — Ctrl+B

  • Italic — Ctrl+I

  • Strikethrough

  • Inline code

  • Link — opens a URL editor inline for the selected text

Attribute Editor

Available on every node at depth > 0. Opens an inline panel where you can set:

  • style — inline CSS with a dedicated style editor

  • class — CSS class list

  • id — element ID

  • Component-specific props — gap, padding, variant, split ratio, etc., shown as dropdowns for enum values

Table Controls

Appear only when the cursor is inside a table cell:

  • Add row before / after

  • Delete current row

  • Add column before / after

  • Delete current column

List Controls

Appear when the active node is a list wrapper:

  • Toggle bullet list

  • Toggle ordered list

  • Toggle task list

  • Switching between types rewrites item node types and preserves content

Media Controls

Shown when an image or video is selected:

  • Image URL editor — edit the src attribute of the selected image

  • YouTube URL editor — edit the video URL and set a responsive max-width

  • Format code — shown for code blocks; runs the formatter for JS and HTML

Node Path & Depth Control

The Node Path Breadcrumb

At the top of the editor, a breadcrumb trail shows the full nesting path of the cursor's current position — for example: doc › LayoutCard › p. Each segment represents one ancestor node, from the document root on the left to the deepest selected node on the right.

Clicking a segment sets the active depth to that level. This controls which blocks show a drag handle. It's the primary way to target a specific nesting level for drag-and-drop when blocks are deeply nested.

Active Depth & the Drag Handle

The active depth is a ceiling: any block at depth ≤ active depth gets a drag handle. A small number above the handle shows its depth. This means:

  • Depth 1 — only top-level blocks show handles (default)

  • Depth 2 — top-level and their direct children get handles

  • Deeper — more nesting levels become individually draggable

The floating toolbar also respects active depth: it targets the shallowest relevant ancestor of the cursor position, up to the active depth ceiling.

Drag Handle & Multi-select

Drag Handle

A ⠿ handle appears to the left of any draggable block when you hover near it. Drag it to reorder. The handle fades after 2 seconds of inactivity — move the cursor back to the left edge to show it again.

Which blocks show handles is determined by the active depth (above). At depth 1 only the outermost blocks are draggable; increasing depth makes nested blocks draggable too.

Multi-select (Virtual Selection)

The toolbar's selection mode lets you select multiple blocks. Use Add siblings or Add children to extend the selection, then:

  • Move before / after target — repositions the whole group

  • Wrap selected — wraps them all in a new container

  • Delete selected — removes all at once

Dragging a node that's part of a multi-select moves the entire group. A composite preview shows up to three cloned nodes with a badge indicating how many are moving.

Inline vs Block Content

Every element in the editor's schema is configured to hold either block content (paragraphs, headings, other layout blocks) or inline content (text, bold, links). A single element cannot mix both — this is a ProseMirror constraint.

If you place a block element (e.g. a paragraph) inside a component that only accepts inline content, it is silently dropped. Conversely, bare text inside a block container is automatically wrapped in a paragraph. The Wrap In toolbar action always targets block-content containers, so inline-only components don't appear in its picker.
